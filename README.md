# Zomato-Restaurant-Analytics-Project
This project analyzes 9,551 restaurants across multiple cities using SQL and MySQL. The goal is to identify customer preferences, restaurant performance, cuisine trends, and value-for-money opportunities through data-driven analysis.

**Dataset**
Source: Zomato Multi-City Restaurant Dataset: https://www.kaggle.com/datasets/shrutimehta/zomato-restaurants-data
Records: 9,551 restaurants
**Key Fields:**
Restaurant Name
City
Cuisines
Average Cost for Two
Aggregate Rating
Votes
Online Delivery
Table Booking

**Business Objectives**
1) Identify cities with the highest restaurant concentration.
2) Measure customer engagement across markets.
3) Discover top-performing cuisines.
4) Analyze the impact of online delivery.
5) Rank restaurants within cities.
6) Identify value-for-money restaurants.
7) Compare city-level restaurant ecosystems.

## SQL Analysis

- Data Exploration
- City Analysis
- Rating Analysis
- Cost Analysis
- Delivery Analysis

**Key Findings**
1) Customer ratings vary significantly by cuisine.
2) Online delivery is associated with higher engagement.
3) Certain cities dominate restaurant activity.
4) Several affordable restaurants achieve premium ratings.
5) Missing geolocation data existed in approximately 5% of records.

**Project Limitations**
1) Snapshot dataset rather than time-series data.
2) No revenue information available.
3) Votes represent engagement but not sales.
4) Some restaurants have missing coordinate data.
